<h1>Undefined Behavior</h1> <p>Keid is designed so that it is impossible for the compiler to output any code with undefined behavior. Any bug that causes this to happen should be reported!</p> <p>If you&#39;re not familiar with what undefined behavior is, <a href="https://en.wikipedia.org/wiki/Undefined_behavior">this Wikipedia page</a> on the topic is a useful introduction.</p> <p>As a short summary, undefined behavior (UB) is behavior that is not defined by the specification of the language. (This definition is simplified for the sake of brevity here.) For example, booleans in Keid are defined to either have a value of <code>0</code> or <code>1</code> and that any other value is <em>undefined behavior</em>. <code>2</code> is not true or false, and any boolean operations or if statements using the value will have unpredicatable <em>undefined behavior</em>.</p> <p>The Keid compiler makes a guarantee to the programmer that it will never output code with undefined behavior. With that being said, programmers can hack and circumvent this; perhaps with <em>unsafe</em> pointer operations. It is the responsibility of the programmer to ensure that their <em>unsafe</em> code does not actually cause UB.</p>
